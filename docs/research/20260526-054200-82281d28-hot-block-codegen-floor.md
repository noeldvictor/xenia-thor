## Findings

The mapped hot block is `82281D28:8228233C-82282370`. It is scalar call and
stack-argument setup, not VMX/vector/GPU work:

- PPC ops: `ld:6`, `addi:3`, `bl:2`, `or:2`, `stw:1`
- HIR ops: `store_context:13`, `load_offset.1:6`, `load_context:5`,
  `add:2`, `call:2`, `context_barrier:2`, `set_return_address:2`,
  `store_offset.1:1`, `truncate:1`
- Context loads: `r[1]:2`, `r[11]:1`, `r[21]:1`, `r[30]:1`
- Context stores: `lr:2`, `r[3]:2`, `r[4]:2`, `r[5]:2`, `r[6]:1`,
  `r[7]:1`, `r[8]:1`, `r[9]:1`, `r[10]:1`
- Calls: `0x826BF770` and recursive `0x82281D28`
- Source-floor estimate: `37` estimated A64-ish lowering instructions, with
  `load_offset.1` and context traffic dominating the block shape.

The body profile row reports `body_ticks_total=44572243`,
`body_ticks_delta=16609748`, `entries_delta=23493`, and `ticks_per_entry=707`.
The report has no call-edge rows for this capture, so the exclusive percentage
is useful for local block ranking only, not for claiming callee-excluded cost.

Literal A64 machine-code disassembly was not available in the capture:
`A64Assembler: ARM64 Capstone disassembly is unavailable`. This is therefore a
HIR/source codegen-floor audit, not a machine-disassembly audit.

Source anchors:

- `LOAD_CONTEXT_I64` / `STORE_CONTEXT_I64` lower to direct PPC-context
  `ldr` / `str` paths in `src/xenia/cpu/backend/a64/a64_sequences.cc`.
- `LOAD_OFFSET_I64` computes a guest address through `x0`, adds membase, and
  loads from host memory in `src/xenia/cpu/backend/a64/a64_seq_memory.cc`.
- Stackpoint synchronization records host/guest stackpoint metadata in
  `src/xenia/cpu/backend/a64/a64_emitter.cc`; it does not remove these guest
  stack loads.

## Decision

Close VMX128, GPU/Vulkan offload, local store elision, and generic fast-entry
behavior for this hot block without fresh evidence. The block is a guest stack
argument handoff shape:

1. Load guest stack arguments from `r1 + constant` via `LOAD_OFFSET`.
2. Store those values into `r3-r10` PPC-context argument registers.
3. Immediately make a direct guest call.

The next useful slice is a default-off counter-only guest-stack argument
handoff audit for direct guest calls. It should count `LOAD_OFFSET` from the
current guest stack pointer feeding `STORE_CONTEXT r3-r10/lr` before direct
guest calls, with target/callsite rows, offset lists, body-time weighting,
compiled/resolved state, context-barrier and stackpoint/debug/exception
blockers, helper/extern/tail blockers, normal-entry fallback pressure, and
estimated avoidable load/store traffic.

Do not run a quiet speed A/B and do not patch behavior from this evidence. If
the stack-argument handoff audit does not show broad route-weighted volume with
a normal-entry-compatible implementation path, close the lane and move to
fastmem/addressing counters for guest stack `LOAD_OFFSET` / `STORE_OFFSET`
traffic.
