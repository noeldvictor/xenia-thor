## `822825E0`

HIR shape:

- PPC: `lwz r3, 0x38(r31)`, `cmpli r3, 0`, `beq 0x822825F4`,
  `or r4, r29, r29`, `bl 0x82282490`
- HIR: `19` instructions, `2` context loads, `6` context stores, one memory
  load, one conditional branch, one direct call, two barriers.
- State touched: loads `r[31]` and `r[29]`; stores `r[3]`, `r[4]`, `lr`, and
  CR0 compare bytes.

Interpretation:

`822825E0` is a child-pointer guard. If `*(r31 + 0x38) != 0`, it recursively
calls `82282490` with:

- `r3 = *(r31 + 0x38)`
- `r4 = r29`

The measured `body_total=34726883` is very unlikely to be the guard itself.
It is the recursive child call path being charged to the caller block.

## `822825C8`

HIR shape:

- PPC: `lwz r4, 0(r31)`, copies `r29` to `r6`, copies `r30` to `r3`, builds
  `r11` with `rlwinm r11,r4,6,0,25`, computes `r5 = r11 + r28`,
  then `bl 0x8227FEE8`.
- HIR: `22` instructions, `4` context loads, `6` context stores, one memory
  load, one direct call, one barrier.
- State touched: loads `r[28]`, `r[29]`, `r[30]`, `r[31]`; stores `r[3]`,
  `r[4]`, `r[5]`, `r[6]`, `r[11]`, and `lr`.

Interpretation:

This block is a low-entry, high-cost call setup:

```text
r4 = *(r31 + 0)
r6 = r29
r3 = r30
r5 = r28 + ((r4 << 6) & 0xFFFFFFC0)
call 0x8227FEE8
```

The `body_peak_tpe=500` is a strong hint that the cost is callee/body work or a
guest-call boundary, not the few local integer operations.

## Decision

Do not start the old `822824F0` `stvewx` peephole from entry-count evidence.
The current body-time evidence says the next real target is call-edge isolation:

1. Add a default-off direct-call edge/body profiler that can charge
   `822825E0 -> 82282490` and `822825C8 -> 8227FEE8` separately, or
2. run a control-sandwiched callee body-time/disassembly capture for
   `8227FEE8` after proving the same route reaches the opening scene.

The first option is the better next automation slice because it avoids another
timing-sensitive route attempt without better attribution.
