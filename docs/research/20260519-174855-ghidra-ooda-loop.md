a broad, slow "analyze the whole disc" detour.

## Decisions

- All new project skills must be repo-local under `.agents/skills/`.
- Create a repo-local Ghidra OODA skill.
- Use ignored `scratch\ghidra\` for native projects, guest PPC/XEX scratch,
  generated JIT blobs, static-analysis reports, and helper scripts.
- Do not commit game files, extracted XEX files, Ghidra projects, title keys,
  account data, screenshots, or generated analysis artifacts.
- Use runtime evidence first: A64 speed-profile top PCs, logcat crashes,
  focused PPC dumps, generated-code ranges, APK hashes, and screenshot route
  proof.


The skill defines the faster loop:

1. Runtime capture produces concrete addresses.
2. Build a tiny address queue.
3. Classify each item as guest PPC, native ARM64, or generated JIT.
4. Import the smallest useful binary into Ghidra.
5. Write a dated triage report with likely owner subsystem and one next
   experiment.
