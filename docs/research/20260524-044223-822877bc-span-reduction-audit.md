## Decision

Do not patch parent-side `822877BC-82287B38` generated code yet.

The parent block is body-backed, but the lower-noise call-edge capture explains
almost all of that block body as the child call edge `82287788 -> 821CE028`:

- parent span body: `1173620`
- child edge body: `1147798`
- approximate parent-exclusive body: `25822` (`2.2%`)

That makes the visible CR/state/vector traffic a misleading next patch target.
The largest scalar-looking lane is CR6 compare/store/barrier traffic, but CR
compare/barrier fusion and CR-store elision are already known crash/regression
lanes for this title. The vector join/store shape is real, but it is not
exclusive-body-proven enough to justify a new default-off toggle before the
callee is understood.
