# 82282598 -> 82287788 State Roundtrip Audit

Date: 2026-05-24 09:51 EDT

## Goal

After the HIR dead-state store DCE audit proved that the exact `13`
candidate-dead call-boundary stores do not recursively delete upstream work,
audit the live state passed from Blue Dragon `82282490` at call PC `82282598`
into callee `82287788`.

This slice is offline-only. It adds deterministic reporting before any generated
code change.

## Commands

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File tools\thor\thor_hir_interproc_state_roundtrip_audit.ps1 -ParentLogPath scratch\thor-debug\20260521-170941-speed-logcat.txt -ParentFunction 82282490 -ParentCallPc 82282598 -CalleeLogPath scratch\thor-debug\20260524-041413-speed-logcat.txt -CalleeFunction 82287788 -EdgeProfileLog scratch\thor-debug\20260524-030450-speed-logcat.txt -Offsets "56,296,2628,16" -Top 24
```

## Tool Added

- `tools/thor/thor_hir_interproc_state_roundtrip_audit.ps1`

The tool parses filtered parent/callee HIR dumps, maps target context offsets to
PPC state fields, finds the last parent stores before or at the direct call,
counts callee loads/stores, and optionally joins matching dynamic call-edge
rows.

Default offsets:

- `56`: `r[3]`
- `296`: `f[1]`
- `2628`: `fpscr`
- `16`: `lr`

## Dynamic Edge

From `scratch\thor-debug\20260524-030450-speed-logcat.txt`:

- edge: `82282490` block `8228252C` -> `82287788`
- `calls_total=1691272`
- `calls_delta=170343`
- `body_ticks_total=5653971`
- `body_ticks_delta=782513`
- `ticks_per_call=4`

This is a high-frequency call boundary. Per-call state traffic is small, but it
is multiplied hard enough to matter.

## Parent Stores

Parent `82282490` stores these tracked context fields before or at call PC
`82282598`:

- `r[3]` (`+56`) at PPC `82282550`
- `f[1]` (`+296`) at PPC `82282594`
- `fpscr` (`+2628`) at PPC `82282594`
- `lr` (`+16`) at PPC `82282598`

The `f[1]` and `fpscr` stores are immediately before the direct call and are not
candidate-dead stores.

## Callee Access

Callee `82287788` loads the same fields:

- `r[3]`: `loads=12`, `stores=15`, first load at `8228779C`
- `f[1]`: `loads=10`, `stores=0`, first load at `82287798`
- `fpscr`: `loads=26`, `stores=26`, first load/store at `82287A1C`
- `lr`: `loads=2`, `stores=5`, first load at `82287788`

The focused callee HIR therefore confirms that `f[1]` and `fpscr` are live
across the call. Skipping the parent stores would hide guest-visible PPC state.

## Decision

Do not skip live parent stores for `f[1]`, `fpscr`, `r[3]`, or `lr`.

Do not implement a HIR-level version of the prior `13` candidate-dead store
suppression. The DCE audit already showed `dead_assignments=0` for that exact
set.

The next useful experiment is a default-off function-pair design for
`82282490 -> 82287788` that keeps PPC state visible while reducing redundant
roundtrips, for example:

- a direct-call carrier for `f[1]` / `fpscr` with explicit flush rules;
- a callee-local promotion probe that reloads once and avoids repeated internal
  context traffic;
- a direct-call argument lowering audit that proves equivalence before a speed
  run.

Any behavior patch must be Blue-Dragon/function-pair gated, default-off, and
route-proven before quiet A/B.

## Next

Design or audit a guarded `82282490 -> 82287788` state carrier / callee-local
promotion lane. The patch target should reduce live state roundtrip traffic,
not delete stores that the callee demonstrably consumes.
