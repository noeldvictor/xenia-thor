## Decision

The audit does not justify a broad VMX rewrite. It does make `822824F0`
`stvewx` / dynamic `extract` the first plausible vector-codegen micro-target,
because that exact shape is both dynamic-hot and mechanically expensive on A64.

Do not start the peephole from entry counts alone. The next slice should add
lower-noise per-block body-time attribution for `82282490`, ideally enough to
separate `822824F0` from the other entry-hot control blocks. If `822824F0`
remains body-time hot, the first codegen experiment should be default-off and
function/block-filtered around the `stvewx` dynamic extract/store shape.
