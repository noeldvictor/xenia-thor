## Source Check

The profiler rows are generated from
`src/xenia/cpu/backend/a64/a64_backend.cc`, which logs
`profile_block_address(i)`.

`src/xenia/cpu/backend/a64/a64_emitter.cc` fills that address through
`FindBlockGuestAddress(block)`, which returns the first `SOURCE_OFFSET` in a
block before falling back to `GuestAddressFor()`. This explains why the runtime
block profile can report an address such as `8228233C` that is not printed as a
HIR label or PPC comment in the dump.

## Decision

Do not patch behavior from weighted HIR/block-profile joins for `82281D28`.
The current join is unsafe: runtime block profile addresses and printed OptHIR
labels/comments do not share a stable key, and ordinal fallback maps hot rows to
different HIR labels.

The previous VMX128 weighted route counters for `82281D28` should be treated as
rough static shape evidence only, not as route-weighted proof for behavior
changes.
