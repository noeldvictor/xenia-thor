# 82287788 Pair-Entry ROI Check

Date: 2026-05-24 11:36 EDT

## Question

Should the next behavior patch be a default-off pair-specific `f[1]` entry for
the hot `82282490:82282598 -> 82287788` edge?

## Findings

The pair-entry candidate is still statically clean:

- `parent_seed status=present pc=82282594`
- `callee loads=10`
- `callee stores=0`
- `replaceable_loads=10`
- `unsafe_loads=0`
- raw `static_replace_upper=16912720`

But the existing stack-slot carrier already covers most of that shape:

- `existing_seed_context_loads=1`
- `existing_reuse_loads=9`
- `pair_incremental_context_loads=1`
- `pair_incremental_static_upper=1691272`

So a pair-specific entry seed would mainly remove the first context load per
`82287788` call, while adding new machinery:

- pair-specific call or thunk;
- callee entry/prolog seed handling;
- normal-entry preservation;
- fallback/kill rules for unknown calls, exits, exceptions, and overlapping
  writes.

## Decision

Do not implement the `f[1]` pair-entry behavior patch as the next immediate
speed experiment by itself.

The idea is valid, but its incremental value over the already-tested stack-slot
carrier is too small for the required call/entry complexity. The previous
quiet A/B already showed that the larger `f[1]` carrier lane was route-clean
but not speed-positive. A pair-entry seed can be revisited if a broader
state-cache design needs this call boundary anyway.

## Next Experiment

Move away from narrow `f[1]` carrier work for the immediate sprint.

Better next lanes:

- design/audit a broader state-cache or callee-variant model for
  `8228252C-822825C4`, not just `f[1]`;
- build an `fpscr` dirty-cache audit that proves exact writeback points before
  any behavior change;
- return to the larger local `82282490` span only if a new audit shows a
  multi-field reduction, not a one-load seed removal.

Keep `arm64_blue_dragon_f1_carrier_fastpath`,
`arm64_blue_dragon_f1_carrier_audit`, pair-entry behavior, `fpscr` forwarding,
broad GPR caches, CR fusion/elision, and stale broad vector peepholes
default-off/out of presets.
