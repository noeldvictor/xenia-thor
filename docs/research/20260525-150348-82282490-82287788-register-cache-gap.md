## Decision

No generated behavior patch from this slice.

The route has enough measured state traffic to justify a mature guest-state
register/cache design, but the evidence says it must be structural and
visibility-aware. The next patch should not be another isolated `load_context`
fold. It should be one of:

1. a default-off guest-state register-cache audit/design skeleton at the
   HIR/A64 register-allocation boundary, with counters for hits, flushes,
   spills, helper/call kills, and normal fallback; or
2. caller-local / side-table edge-variant payload storage for
   `82282490:82282598 -> 82287788`, with normal-entry fallback and explicit
   payload materialization counters.

Keep fpscr behavior unchanged, keep narrow stvewx / `f[1]` carrier probes
default-off, and keep Vulkan as a parallel lane only until captures show GPU
work overtaking Main Thread / A64 generated code.
